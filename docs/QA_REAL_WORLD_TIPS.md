# Real QA Tips (현업 QA 꿀팁)

다음 팁들은 실제 QA 업무에서 자주 사용하는 테스트 사고 방식입니다.  
짧은 문장이지만 QA 테스트 설계와 버그 발견에 매우 중요한 기준이 됩니다.

---

1. 정상 케이스보다 예외 케이스에서 버그가 더 많이 발생합니다.

2. 입력 필드가 있다면 공백, 특수문자, 매우 긴 문자열을 항상 테스트해보세요.

3. 숫자 제한이 있다면 항상 경계값(최소값, 최대값)을 확인하세요.

4. 버튼은 한 번만 누른다고 가정하지 말고 연속 클릭도 테스트하세요.

5. 같은 기능을 여러 번 반복 실행하면 예상하지 못한 버그가 발생할 수 있습니다.

6. 상태가 바뀌는 기능은 모든 상태에서 테스트해야 합니다.

7. 로그인, 결제, 저장 기능은 네트워크가 끊겼을 때도 확인해야 합니다.

8. 기능이 정상 동작하더라도 데이터가 제대로 저장되는지 확인해야 합니다.

9. 에러 메시지는 사용자에게 이해 가능한 문장인지 확인해야 합니다.

10. 기능을 테스트할 때는 항상 "사용자가 이상하게 사용하면 어떻게 될까?"를 생각하세요.

11. QA는 UI가 아니라 시스템 동작을 검증하는 직무입니다.

12. 재현 경로가 명확하지 않은 버그 리포트는 개발자가 수정하기 어렵습니다.

13. 좋은 버그 리포트는 누구라도 동일하게 재현할 수 있어야 합니다.

14. 테스트 케이스는 기능을 이해하는 과정이기도 합니다.

15. 같은 기능이라도 다른 환경(브라우저, OS, 모바일)에서 결과가 달라질 수 있습니다.

16. QA는 버그를 찾는 직무가 아니라 제품 품질을 지키는 직무입니다.

17. 버그를 발견했다면 왜 발생했는지 추측해보는 습관이 중요합니다.

18. 기능이 정상처럼 보여도 데이터 흐름이 잘못된 경우가 많습니다.

19. 작은 UI 오류라도 사용자 경험에 큰 영향을 줄 수 있습니다.

20. QA의 가장 중요한 능력은 "의심하는 사고방식"입니다.
